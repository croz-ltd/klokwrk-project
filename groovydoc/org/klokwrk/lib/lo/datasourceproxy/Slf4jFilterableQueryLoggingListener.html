class Slf4jFilterableQueryLoggingListener
extends net.ttddyy.dsproxy.listener.logging.SLF4JQueryLoggingListener

          Extends DataSourceProxy's net.ttddyy.dsproxy.listener.logging.SLF4JQueryLoggingListener by adding capability of filtering out queries based on regular expression match.

          If needed, filtered out queries can still be sent to log if the logging level is set to TRACE for the appropriate logger.

          Filtering queries might be very useful when there is some kind of recurring queries happening, for example when we have some kind of database polling. One good example is Axon queryside projection application, where projection is projecting into RDBMS database. In that scenario, Axon continually issues update token_entry statements for maintaining token_entry table. Commonly we do not want to see these statements all the time and it is convenient to be able to filter them out.
